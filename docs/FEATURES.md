## Files, classes, and storage

**File-first model.** Files (blobs) are the primary entity; a **class** is a schema + opt-in per-blob metadata. The full design is [`docs/FILE_FIRST_CLASSES.md`](FILE_FIRST_CLASSES.md). On disk:

```
<root>/media/{ manifest.json, files/, classes/<id>.json, settings.json }   # the hub
<root>/<jsonType>/{ settings.json, data.json }                             # `json` types (incl. globals)
```

| Concept     | Meaning                                                                              | Storage                                                   | Repo                                              | UI                                                                                     |
| ----------- | ------------------------------------------------------------------------------------ | --------------------------------------------------------- | ------------------------------------------------- | -------------------------------------------------------------------------------------- |
| Blob        | A file in `media/files/`, identity = manifest `file_id` (UUID)                       | [`manifest.ts`](../src/lib/storage/manifest.ts)           | [`classRepo.ts`](../src/lib/storage/classRepo.ts) | All Files hub ([`files/+page.svelte`](../src/routes/files/+page.svelte))               |
| Class       | Schema + opt-in metadata keyed by `file_id` (any file type; empty schema = pure tag) | `media/classes/<id>.json` = `{ schema, config, records }` | [`classRepo.ts`](../src/lib/storage/classRepo.ts) | Hub sidebar + [`FileEditorPanel.svelte`](../src/lib/components/FileEditorPanel.svelte) |
| `json` type | Records only, no file attachment (`records/` reorg deferred — Item 18)               | `<root>/<typeId>/data.json`                               | [`jsonRepo.ts`](../src/lib/storage/jsonRepo.ts)   | [`JsonEditorPane.svelte`](../src/lib/components/JsonEditorPane.svelte)                 |

**Membership is opt-in and binary (by design):** a blob is a **member** of a class iff that class file has a record keyed by its id; **unclassified** = empty `classes[]`. There is **no** linked/unlinked/excluded tri-state, no `excludedFiles`, and **no per-extension gating** — any blob can belong to any class. The manifest's `classes[]` is a **derived cache**; class files are the source of truth. Consequences (all expected):

- **Add/remove to class** ([`addMembers`/`removeMembers`](../src/lib/storage/classRepo.ts)) writes the class file and updates `manifest.classes[]` atomically. Bulk add creates empty member records (fields filled inline later).
- **mtime-gated resync:** on list, if any class file is newer than the manifest, the index is rebuilt from class files ([`applyMembershipIndex`](../src/lib/storage/manifest.ts)) — catches hand edits.
- **Lazy heal:** every list reconciles the manifest against disk ([`reconcile`](../src/lib/storage/manifest.ts)) — new blobs get an id, vanished blobs are flagged `missing: true` (kept). Response carries `healed: { added, missing }`.
- **Rename** is **O(1)** ([`renameBlobById`](../src/lib/storage/classRepo.ts) → [`manifest.renameFileId`](../src/lib/storage/manifest.ts)) — disk + one manifest entry, no fan-out.
- **Delete from disk** drops the manifest entry and strips the member record from **every** class ([`deleteFromDiskById`](../src/lib/storage/classRepo.ts)). **Delete class** removes the file + strips its id from `classes[]`; blobs untouched.
- **Locking:** manifest mutations are atomic under one `.lock`, taken **before any class lock**.

**Filenames:** blobs use [`assertSafeBasename`](../src/lib/storage/filenames.ts) (any extension). Intrinsic image dims are backfilled into manifest entries on listing.

**Root discovery:** `MEDIA_MANAGER_ROOT` → [`getRootDir()`](../src/lib/storage/paths.ts). `listClassIds()` scans `media/classes/`; `listMediaTypeIds()` scans the root for `json` types (skipping `media/`).

**Reserved type ids:** only `globals` ([`RESERVED_TYPE_IDS`](../src/lib/storage/mediaTypes.ts), auto-created/healed by `ensureGlobalsGroupExists()`) — protected from create/rename/delete. The old `files` blob_store type is gone (`media/` is the hub). No reserved classes.

**Missing files:** a `file`-field value (or a class membership) referencing a blob absent from disk is surfaced globally via `GET /api/files/missing` ([`listMissingFileReferences`](../src/lib/storage/classRepo.ts)) and the hub's warning banner; class list items still carry `missing_file_fields: string[]` and records carry `_missing_files: { field → expected filename }`.

## HTTP API index (quick reference)

### Files / blobs (`/api/files/...`)

| Path pattern                                | Role                                                                                                                               |
| ------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| `GET /api/files`                            | All Files hub list (`FileListResponse`; `?query&classIds&match=all\|any&unclassified&groupByClass&groupByField`; carries `healed`) |
| `POST /api/files/upload`                    | Multipart upload (any type; HEIC→JPEG); registers the blob, returns `id`                                                           |
| `GET /api/files/[id]/blob`                  | Raw blob bytes                                                                                                                     |
| `POST /api/files/[id]/rename`               | Rename blob on disk + manifest (O(1)); body `{ new_filename }`                                                                     |
| `POST /api/files/delete`                    | Bulk delete from disk; body `{ ids }` (strips member records across classes)                                                       |
| `GET /api/files/[id]/metadata`, `.../strip` | EXIF read / strip (`{ mode: 'all'\|'gps' }`)                                                                                       |
| `GET /api/files/[id]/classes`               | Per-file editor sections: each class this blob belongs to + its schema/config/record (also the delete-usage surface)               |
| `GET /api/files/missing`                    | Global missing-files warning: dangling refs grouped per absent blob                                                                |

### Classes (`/api/classes/...`)

| Path pattern                                   | Role                                                                                   |
| ---------------------------------------------- | -------------------------------------------------------------------------------------- |
| `GET/POST /api/classes`                        | List classes (+counts) / create (`{ displayName, schema? }`)                           |
| `GET/PATCH/DELETE /api/classes/[id]`           | Read schema+config / update config (rename, grid prefs) / delete class                 |
| `GET/POST/DELETE /api/classes/[id]/members`    | Catalog view (`FileListResponse`, `?groupBy&query`) / add / remove members (`{ ids }`) |
| `/api/classes/[id]/schema`                     | Schema CRUD: `GET` read, `POST` add, `PATCH` update, `DELETE` remove, `PUT` replace    |
| `GET/PATCH /api/classes/[id]/records/[fileId]` | One member's record (carries `_missing_files`) / update fields                         |
| `POST /api/classes/[id]/records/bulk-update`   | Apply one patch to many members (`{ ids, patch }`)                                     |
| `GET /api/classes/[id]/field-values`           | Distinct values for autocomplete (`?field`)                                            |

### `json` media types (`/api/media-types/...`)

| Path pattern                                                               | Role                                                       |
| -------------------------------------------------------------------------- | ---------------------------------------------------------- |
| `GET/POST /api/media-types`                                                | List `json` types / create (`{ displayName }`)             |
| `/api/media-types/[typeId]`                                                | Get / patch display name / delete                          |
| `.../stats`, `.../schema`, `.../settings`                                  | Counts+mtime / schema CRUD / type settings (display name)  |
| `.../records/list`, `.../records`                                          | List (`JsonListResponse`) / create record                  |
| `.../records/field-values`, `.../records/by-id/[id]`                       | Autocomplete / get·patch·delete record                     |
| `.../records/bulk-update`, `.../records/bulk-delete`, `.../records/repair` | Bulk patch / bulk delete / schema repair                   |
| `/api/media-types/globals/record`                                          | Read/update singleton globals object (`null` removes key)  |
| `GET`/`POST /api/settings`                                                 | Global app prefs (grid size, nav) in `media/settings.json` |

Legacy `/api/images/*`, `/api/schema`, `/api/config` were **removed** — migrate via `upgrade-data`.

### Client wrappers

Files/classes: [`src/lib/api/files.ts`](../src/lib/api/files.ts). `json`-side `*ForType` helpers: [`src/lib/api/client.ts`](../src/lib/api/client.ts).

## Residual notes

- **README** is the canonical place for install, **build + run** (`npm run build`, `node build`, `media-manager <root>`), and data layout examples.
- **`docs/FUTURE_CHANGES.md`** is the deferred-work backlog (non-committing ideas; do not treat it as the shipped feature set). `tasks.md` has been consolidated into it and is now just a pointer.
- **Data upgrade tool:** [`scripts/upgrade-data.mjs`](../scripts/upgrade-data.mjs) (`npm run upgrade-data -- <root>` or `... -- <root> --apply`) scans a data root and reports/applies the expected upgrades, culminating in the **file-first reorg** (step 5): relocate `files/` → `media/files/`, rebuild the manifest at `media/manifest.json` (v2, with derived `classes[]`), and convert each `images`/`generic` catalog folder → `media/classes/<id>.json` (`{ schema, config, records }`, dropping `image_name`/`excludedFiles`); the `blob_store` `files` type dissolves; `json`/`globals` folders stay. Idempotent (skipped once `media/classes` exists), dry-run by default.
