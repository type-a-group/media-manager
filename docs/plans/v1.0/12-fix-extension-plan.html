Item 12 — Fix / Normalize File Extension (+ better rename UX)

v1.0 brief · file-first model · Phase 2 plan (approve before build). Backlog: FUTURE_CHANGES.md Item 12 · Brief: 12-fix-extension.md.

interview donesize Sno new dep

1 · What this solves

A file's identity is its manifest id, not its name — so renaming (full name or just the extension) is an O(1) display-name change via renameBlobById, with zero fan-out to class memberships. Two jobs:

  • Fix a wrong extension. A file whose bytes are a PNG but whose name ends .jpg should be easy to correct to the sniffed type, in one tap.
  • Make renaming first-class. Split the editor's filename into a base-name field and a small extension field so an extension fix is one interaction, and consolidate today's two competing rename paths into one.

3 · Decisions (locked in interview)

  • Surfacing → proactive badge. Mirror Item 13: an amber “!” badge on the Metadata button whenever the extension disagrees with sniffed content (cheap — 12 magic bytes, no exiftool). The detailed fix stays in the popup banner.
  • Rename UX → split base-name + extension fields in the editor header, with an inline detected: .png · Fix hint when the sniff disagrees. Consolidate the duplicate rename paths into this one.
  • Cut line: no auto-fix on upload (always user-confirmed); no bulk extension-fix (per-file only).
  • Bonus (in scope): add cheap GIF / TIFF / BMP magic bytes to the sniff table (left un-cut in the interview).
