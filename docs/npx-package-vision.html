media-manager as an npx package

Refined 2026-06-22 · companion to docs/plans/npx-package-vision.md · no code yet

Run npx media-manager inside a host repo (e.g. nicb.at) and have it find the right folder with zero config, plus a read-only path for the host build to consume the on-disk format. This pass is grounded in a read of the current code — two of the original "open questions" turned out smaller than assumed.

Decisions locked this pass

Decided Editor (Mode A)

  • Quiet heal is mostly done. reconcile already writes only if (changed) — browsing committed data is zero-write. #1 shrinks to write-format stability.
  • Ephemeral port: the CLI picks a free port (net…listen(0)), passes it to the child, opens that exact URL. No PORT=0, no 2s race.
  • Distribution: local dep in nicb.at now; scoped publish / npm link later.
  • Config: standalone media-manager.config.json, root only.

Design-only Reader (Mode B)

  • Build nothing yet — refine the design until nicb.at's needs are concrete.
  • Shape: library-first (media-manager/fs); export JSON is a thin wrapper over it.
  • Blobs: point the host static dir at media/files/ (pattern 1); copy-into-build deferred.
  • Gating refactor: thread an explicit root through the read paths — deliberately not started.

The two modes (don't conflate them)

Mode A Editor

npx media-manager → Node server + browser

  • Mutates the root: heals manifest, can rename/strip blobs
  • Interactive curation of files & metadata
  • Real change → one minimal manifest diff (format-stable)

Mode B Reader

import 'media-manager/fs' · media-manager export

  • Never writes, never heals — frozen contract
  • Host build renders galleries/pages from the same files
  • Reports staleness; does not "fix" it

Mode A — zero-config root resolution

First match wins. Today only steps 1–2 exist; 3–5 are the vision.

  1. Explicit arg — media-manager ./root (current)
  2. Env — MEDIA_MANAGER_ROOT (current)
  3. Config file — nearest media-manager.config.json / .media-managerrc / "mediaManager" key in package.json, found by walking up
  4. Convention auto-detect — find a folder with media/manifest.json; probe ./media_manager, ./src/assets/media_manager, ./media
  5. Friendly error — print the chain tried + suggest media-manager init

Command Purpose Status

doctor [root] Validate / report healed & missing without serving later
export … Mode B — emit denormalized JSON snapshot design-only

#1 Quiet heal — already write-guarded

Premise corrected: reconcile (manifest.ts) writes only if (changed); so do setClassMembership / applyMembershipIndex. Pure browsing of unchanged committed data is zero-write today — there is no new quiet-heal code path to add.

Residual A — resync churn (CPU only)

The mtime-gated resync re-fires after every git checkout (git drops mtimes), but recomputes membership and skips the write when content matches. Wasted CPU, no diff.

Residual B — format stability the real task

A genuine change rewrites the whole manifest.json. Commit files in the writer's exact byte-format so that one change is a minimal diff, not a whole-file reformat.

#2 Ephemeral port — the CLI owns the port

Today

// bin/media-manager.js
env.PORT = '3000';        // fixed
spawn(node, [build], { env });
setTimeout(() => open(
  `localhost:${PORT}`), 2000); // race

Target — CLI picks a free port

const port = await freePort();   // net…listen(0)
env.PORT = String(port);
spawn(node, [build], { env });
open(`http://localhost:${port}`); // exact, no race

Deterministic, can't collide with the host dev server, and removes the 2-second open-the- browser guess. Chosen over PORT=0 (which would force reading the bound port back out of the child's stdout).

Host buildgalleries, pages, content-hashed assets

Library (the primitive)

import { openWorkspace } from 'media-manager/fs';

const ws = openWorkspace('src/assets/media_manager');
await ws.listFiles({ classIds: ['photos'] });
await ws.listMembers('photos');
ws.resolveBlobPath(id); // abs path on disk
await ws.getGlobals();

Export (thin wrapper)

media-manager export --class photos \
  --out photos.json

media-manager export --all \
  > workspace.json

Just openWorkspace(root).serialize() piped to a file → any host reads it with plain JSON.parse.

Refactor that gates Mode B (deferred): the storage layer resolves the root from getRootDir() (the MEDIA_MANAGER_ROOT env global). Thread an explicit WorkspacePaths struct through the read paths; keep env-reading getRootDir() as the Mode-A default factory so the server is untouched.

Sequencing

  1. Ephemeral portCLI-only, tiny, self-contained. Ships the "no port config" win first.
  2. Write-format stabilityCommit files in the writer's exact format → committed-data diffs disappear.
  3. Root-discovery chain + media-manager.config.json + local-dep wiringThe headline Mode A experience inside nicb.at.
  4. Mode B (deferred)root-threading (#5) → library (media-manager/fs) → export wrapper → blob-serving. Design-only until nicb.at's needs are concrete.

North-star layout in nicb.at

~/Projects/nicb.at/
├─ media-manager.config.json          # { root: "src/assets/media_manager" } — no port (ephemeral)
├─ src/assets/media_manager/          # the workspace (committed to git)
│  └─ media/
│     ├─ manifest.json
│     ├─ files/
│     ├─ classes/
│     └─ settings.json
└─ src/lib/media.ts                   # imports media-manager/fs → renders galleries at build

Curate: npx media-manager from anywhere in the repo → editor opens on the right folder. Build: the site reads the same files via the reader (Mode B, later).

Packaging gotchas measure first — all deferred

Install size

  • exiftool-vendored bundles platform Perl (tens of MB) → maybe optional/lazy
  • Drop sharp + @masonry-grid/svelte (unreferenced in src/)
  • npx refetches the whole tarball on a cold cache

Distribution

  • build/ must ship (it does; prepublishOnly builds it)
  • Name likely taken → @nicbat/media-manager
  • Add engines.node; npm link for local bare bin
