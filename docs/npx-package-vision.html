media-manager as an npx package

Vision diagram — run npx media-manager inside a host repo (e.g. nicb.at) and have it find the right folder, plus a read-only path for the host build to consume the on-disk format. Companion to docs/plans/npx-package-vision.md. No code yet.

The two modes (don't conflate them)

Mode A Editor

npx media-manager → Node server + browser

  • Mutates the root: heals manifest, can rename/strip blobs
  • Interactive curation of files & metadata
  • Launching on committed data can produce git diffs

Mode B Reader

import 'media-manager/fs' · media-manager export

  • Never writes, never heals — frozen contract
  • Host build renders galleries/pages from the same files
  • Reports staleness; does not "fix" it

Mode A — zero-config root resolution

First match wins. Today only steps 1–2 exist; 3–5 are the vision.

  1. Explicit arg — media-manager ./root (current)
  2. Env — MEDIA_MANAGER_ROOT (current)
  3. Config file — nearest media-manager.config.json / .media-managerrc / "mediaManager" key in package.json, found by walking up
  4. Convention auto-detect — find a folder with media/manifest.json; probe ./media_manager, ./src/assets/media_manager, ./media
  5. Friendly error — print the chain tried + suggest media-manager init

Mode B — host reads the format at build time

ReaderopenWorkspace(root) — pure, root-explicit, no env

Host buildgalleries, pages, content-hashed assets

Library (Node hosts)

import { openWorkspace } from 'media-manager/fs';

const ws = openWorkspace('src/assets/media_manager');
await ws.listFiles({ classIds: ['photos'] });
await ws.listMembers('photos');
ws.resolveBlobPath(id); // abs path on disk
await ws.getGlobals();

Export (any host)

media-manager export --class photos \
  --out photos.json

media-manager export --all \
  > workspace.json

Denormalized snapshot → host reads with plain JSON.parse.

Refactor that gates Mode B: the storage layer resolves the root from getRootDir() (the MEDIA_MANAGER_ROOT env global). A library API must thread an explicit root instead of mutating process.env.

North-star layout in nicb.at

~/Projects/nicb.at/ ├─ media-manager.config.json { root: "src/assets/media_manager" } (port is ephemeral) ├─ src/assets/media_manager/ the workspace (committed) │ └─ media/{manifest.json, files/, classes/, settings.json} └─ src/lib/media.ts imports media-manager/fs → renders at build

Packaging gotchas measure first

Install size

  • exiftool-vendored bundles platform Perl (tens of MB) → maybe optional/lazy
  • Drop sharp + @masonry-grid/svelte (unreferenced in src/)
  • npx refetches the whole tarball on a cold cache

Distribution

  • build/ must ship (it does; prepublishOnly builds it)
  • Name likely taken → @nicbat/media-manager
  • Add engines.node; npm link for local bare bin
