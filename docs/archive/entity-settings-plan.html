Unified Entity Settings FINAL · all options chosen

One shared “entity settings” surface for both a Files class and a Records type. This doc reflects the locked decisions; implementation plan lives in docs/plans/entity-settings-implementation.md.

Locked decisions
  • ⋮ menu: Style 1 — one Settings… entry + Delete.
  • Popup: Option A — tabbed dialog (General / Fields / Danger).
  • Rail: shared shell, two modes — Records navigates, Files filters; Files becomes collapsible.
  • Collapsed access: content-header ⋮ (always reachable) + row ⋮ when expanded.
  • Title by: persisted per-entity, removed from the list header. Group by stays an ephemeral header control.
  • Rename the type's display name is a field in the popup. Old ClassSettingsDialog + ClassSchemaDialog are retired into the shared dialog.

In Files' multi-select mode there's no single active class, so the header ⋮ only appears when a single class catalog is open; otherwise you expand the rail. 5 · App settings vs entity settings

Guardrails (every screen)

  • Per-entity dialog title is always {Name} — settings, never bare “Settings”.
  • Footer gear relabeled “App settings” (was “Settings”).
  • Dialog subtitle: “Settings for this {noun} — global preferences live in App settings.”
  • ⋮ lives on the row / in the content header; the gear lives in the footer.

What goes where

App settings (gear) Entity settings (⋮)
Grid / tile size Display name (rename)
Rail collapsed, nav prefs Title by (persisted)
Autosave-on-advance Schema / fields
Global, cross-entity Delete this entity

Group by stays a live header control — a momentary view, not a saved pref.
