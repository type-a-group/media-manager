Unifying the Files & Records interfaces

Two sibling sub-apps — Files (/files, blobs + opt-in classes) and Records (/media, schema-backed types) — should feel like one product. This makes their chrome converge on shared primitives, moves Records' search into the rail, adds a search-field picker to both, hides the editor panel until a row is clicked, and deletes a pile of forked code.

Locked decisions.
  • Header stays in the content column — the panel opens beside it. The "full-width" feel comes for free once Records stops reserving panel space.
  • Maximize reuse — Records' detail pane adopts EditorPanelShell; its list adopts DataGrid (text variant + subtitle); both sides share a new SearchBox and a schemaUserFields() util.
  • Files header ⋮ stays solo-class-only (already correct).
  • Search-field picker defaults to All fields, narrowable to one.

Behavioral reconciliations (the risk in "maximize reuse").
  • Autosave (decided): both sides adopt Records' debounced-always autosave. FileEditorPanel loses its explicit per-section "Save" buttons and gains the debounced save + "Saving…/Saved" status. One save policy across both panels — simpler merge onto EditorPanelShell.
  • Arrow keys: Files uses ←/→, Records' pane used ↑/↓. Unify on EditorPanelShell's ←/→ (skipped while typing in a field).
  • DataGrid subtitle (decided): add an optional secondaryLabel to GridItem now — Records' text list uses it for title + muted subtitle. Wiring a subtitle onto the Files thumbnail tiles is deferred (the field is there, unused on that side for now).
  • Search scope: Records search is client-side over the title today; field-scoped search needs it to match the chosen field's value (client-side filter over the loaded list keeps it simple; no API change required for v1).

4 · Out of scope (for this pass)

  • Globals' bespoke GlobalsEditorPane — untouched.
  • Server-side field search / new API params — v1 filters client-side over the loaded list.
  • Mobile drill-down, deep-link routing, icon picker (existing FUTURE_CHANGES items).
