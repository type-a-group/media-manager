Sub-app restructure: split Globals, add a switcher, breadcrumbs & ⌘K

Promote Globals to a true peer route, make hopping between the three sub-apps a header dropdown, give every screen breadcrumbs + deep-linkable URLs, and add a global command palette. Backend/on-disk model is untouched — this is almost entirely a frontend & routing change.

Zero on-disk change. Globals stays a reserved json singleton at <root>/globals/ with the same /api/media-types/globals/record API. We're only moving where its editor renders in the UI — so test-fixtures/ needs no regeneration.

1 Globals → its own /globals route

Decision: Own top-level route. Globals becomes a peer of Files/Records, not a row in the Records rail.

  • NEW src/routes/globals/+page.svelte — thin page: EntityRail shell (switcher header, optional section-jump body) + GlobalsEditorPane in the content area.
  • EDIT src/routes/media/+page.svelte — drop the isGlobals branch & GlobalsEditorPane import; filter globals out of types; if ?type=globals arrives, goto('/globals') (back-compat redirect).
  • EDIT src/lib/components/records/RecordsRail.svelte — list already excludes the globals ⋮; now it won't receive globals at all (parent filters it).
  • EDIT src/routes/+page.svelte — dashboard Globals card link /media?type=globals → /globals.

GlobalsEditorPane is reused as-is (it already owns its sections, autosave, and settings dialog). The new page just wraps it in the rail shell so the switcher is present.

Optional polish

The /globals rail body can list the editor's sections as jump links (Globals has no entity list to fill the rail otherwise). Nice-to-have, not required for the split.

▶ Phasing & verification

Phase A — Globals split. New /globals route, strip from Records rail, dashboard + redirect. Verify globals editor loads & autosaves at the new URL; /media?type=globals redirects.
Phase B — Switcher. SubAppSwitcher + EntityRail header rework; wire all three hosts (incl. collapsed). Verify hopping Files↔Records↔Globals + active checkmark.
Phase C — Breadcrumbs + deep links. breadcrumb dep, Breadcrumbs wrapper, ?record=/?file= URL sync on both hubs. Verify reload/share/back-forward restores the open item.
Phase D — ⌘K palette. command dep, CommandPalette, global key handler in layout. Verify ⌘K opens and jumps to a class/type/sub-app.
Phase E — Docs. FEATURES.md (route, switcher, deep-link params, palette), CLAUDE.md (frontend section), FUTURE_CHANGES.md (relation field).

Each phase ends with npm run check + npm run lint; end-to-end via the test-ui-feature skill (build → serve test-fixtures/ → drive with Playwright).

! Risks, scope guards & docs

  • Watch: deep-link sync vs. autosave — URL writes must use replaceState and must not trigger list refetches (the established “patch row in place, never refetch on autosave” rule).
  • Watch: ⌘K key handler must not fire while typing in inputs/the palette itself; respect platform (⌘ vs Ctrl).
  • Guard: all new UI composes shadcn primitives (DropdownMenu, Command, Breadcrumb) — no hand-rolled markup (CLAUDE.md non-negotiable).
  • Guard: no storage/API/media-kind change → test-fixtures/ untouched; only doc updates required alongside.
